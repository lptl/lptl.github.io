• child processes and parent processed. fork() to create a child process from parent process. All content, including the executing program is copied from parent process to child process. To distinguish current process, for parent process, fork() returns pid of child process and for child process, fork() returns 0. Usually, the process spawn procedure is used to perform different task. So exec() is needed and usually used on child process. By using exec(), child process would execute another program and copied content will be lost. However, at this moment, the parent process still needs to use wait() to fetch the exit status of child process and handle its memory allocation after the termination of child process.

  • Unix systems have a single directory tree. All accessible storage must have an associated location in this single directory tree. Mounting is the act of associating a storage device to a particular location in the directory tree. Let's say the CD-ROM device is /dev/cdrom and the chosen mount point is /media/cdrom. The corresponding command is mount /dev/cdrom /media/cdrom. After that command is run, a file whose location on the CD-ROM is /dir/file is now accessible on your system as /media/cdrom/dir/file. When you've finished using the CD, you run the command umount /dev/cdrom or umount /media/cdrom (both will work; typical desktop environments will do this when you click on the “eject” or ”safely remove” button).
